2Ô∏è‚É£ URL Routing & Clean URLs

‚ùì Kenapa Sebelumnya Pakai .php, Sekarang Tidak?

Penjelasan Sederhana:

Bayangkan website seperti sebuah toko. Dulu, untuk masuk ke ruangan tertentu, kamu harus menyebut nama lengkap ruangannya termasuk ekstensinya:

  • Dulu: www.polresta.com/berita.php
  • Sekarang: www.polresta.com/berita

Lebih simple kan? Ini seperti kamu cukup bilang "ruang berita" tanpa perlu bilang "ruang berita.php"

üîß Bagaimana Cara Kerjanya?

Flow Diagram - Cara Kerja Clean URL:

STEP 1: User mengetik polresta.com/berita di browser
STEP 2: Request diterima oleh web server Apache
STEP 3: Apache membaca file .htaccess
STEP 4: File .htaccess mengubah URL:
berita ‚Üí berita.php
STEP 5: Apache menjalankan file berita.php
STEP 6: Halaman ditampilkan ke user (tapi URL tetap /berita)

üìÑ File yang Berperan: .htaccess

File .htaccess adalah file konfigurasi Apache yang mengatur "aturan lalu lintas" website. Berikut isi file .htaccess yang dibuat:

RewriteEngine On
RewriteBase /webandruy/

# Matikan MultiViews
Options -MultiViews

# RULE 1: Berita dengan Slug
# URL: /berita/judul-berita-ini
# Diterjemahkan menjadi: berita-detail.php?slug=judul-berita-ini
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^berita/([a-zA-Z0-9_-]+)$ berita-detail.php?slug=$1 [L,QSA]

# RULE 2: Clean URLs untuk halaman umum
# URL: /dashboard
# Diterjemahkan menjadi: dashboard.php
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^([a-zA-Z0-9_-]+)$ $1.php [L,QSA]

# RULE 3: Redirect .php URLs ke clean URLs (SEO)
# Jika user akses: /dashboard.php
# Redirect ke: /dashboard
RewriteCond %{THE_REQUEST} \s/webandruy/([a-zA-Z0-9_-]+)\.php[\s?] [NC]
RewriteRule ^ /webandruy/%1 [R=301,L]

Penjelasan Setiap Baris:

Kode Penjelasan
RewriteEngine On Mengaktifkan mesin rewrite (pengubah URL) Apache
RewriteBase /webandruy/ Menentukan folder dasar website (karena ada di subfolder webandruy)
Options -MultiViews Matikan fitur Apache yang bisa bikin konflik dengan clean URLs
RewriteCond %{REQUEST_FILENAME} !-f Cek: Apakah file dengan nama itu benar-benar ada? (!-f = bukan file)
RewriteCond %{REQUEST_FILENAME} !-d Cek: Apakah folder dengan nama itu ada? (!-d = bukan directory)
RewriteRule Aturan pengubahan URL
[L,QSA] L = Last (aturan terakhir), QSA = Query String Append (pertahankan parameter)
[R=301,L] R=301 = Redirect Permanent (penting untuk SEO!)

‚úÖ Keuntungan Clean URLs:

  • SEO Friendly: Google lebih suka URL yang clean dan readable
  • User Friendly: Lebih mudah diingat dan dibagikan
  • Professional: Terlihat lebih modern dan profesional
  • Keamanan: Tidak expose teknologi yang digunakan (.php)

3Ô∏è‚É£ Slug & SEO (Search Engine Optimization)

üêå Apa itu Slug?

Definisi Sederhana:

Slug adalah versi "URL-friendly" dari judul. Slug mengubah judul berita menjadi format yang bisa digunakan di URL.

Contoh Konversi:

Judul Berita Slug (URL-friendly)
Polres Padang Tangkap Pelaku Curanmor polres-padang-tangkap-pelaku-curanmor
Razia Kendaraan di Jalan Sudirman razia-kendaraan-di-jalan-sudirman
Kapolres: Tetap Waspada COVID-19! kapolres-tetap-waspada-covid-19

üîÑ Proses Pembuatan Slug:

INPUT: Judul Berita = "Polres Padang Tangkap Pelaku Curanmor"
STEP 1: Ubah ke huruf kecil
‚Üí "polres padang tangkap pelaku curanmor"
STEP 2: Hapus karakter khusus (!?.,@#$%)
‚Üí "polres padang tangkap pelaku curanmor"
STEP 3: Ganti spasi dengan tanda hubung (-)
‚Üí "polres-padang-tangkap-pelaku-curanmor"
OUTPUT: Slug = "polres-padang-tangkap-pelaku-curanmor"

üìä Perbandingan URL Lama vs Baru:

Aspek URL Lama (ID-based) URL Baru (Slug-based)
Format /berita-detail?id=123 /berita/polres-tangkap-pelaku
Readability ‚ùå Tidak jelas isinya apa ‚úÖ Langsung tahu isi berita
SEO Score ‚≠ê‚≠ê (2/5) ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê (5/5)
Share di Medsos ‚ùå Terlihat seperti spam ‚úÖ Terlihat profesional
Google Ranking ‚ùå Lebih sulit ranking ‚úÖ Lebih mudah ranking

üéØ Implementasi di Database:

Untuk mendukung slug, kolom baru ditambahkan ke tabel berita:

ALTER TABLE berita 
ADD COLUMN slug VARCHAR(255) UNIQUE AFTER judul;

Penjelasan:

  • slug VARCHAR(255) - Kolom untuk menyimpan slug (maksimal 255 karakter)
  • UNIQUE - Memastikan tidak ada slug yang sama (mencegah konflik URL)
  • AFTER judul - Letakkan kolom slug setelah kolom judul

üîç SEO (Search Engine Optimization)

Apa itu SEO?

SEO adalah teknik untuk membuat website kita mudah ditemukan di Google. Seperti memasang papan nama toko yang jelas dan menarik agar mudah ditemukan orang.

Fitur SEO yang Ditambahkan:

Fitur SEO Fungsi Implementasi
Meta Description Deskripsi singkat yang muncul di hasil pencarian Google Kolom meta_description di tabel berita
Canonical URL Memberitahu Google URL mana yang asli (mencegah duplikat konten) <link rel="canonical"> di HTML
301 Redirect Redirect permanen dari URL lama ke URL baru Di file .htaccess dan PHP
Clean URLs URL tanpa parameter (?id=123) lebih disukai Google Sistem slug + .htaccess

üí° Cara Kerja System Slug di Website:

// File: berita-detail.php

// 1. Terima parameter slug dari URL
$slug = isset($_GET['slug']) ? trim($_GET['slug']) : '';

// 2. Jika ada slug, cari berita berdasarkan slug
if ($slug !== '') {
    $stmt = $conn->prepare("SELECT * FROM berita WHERE slug=? LIMIT 1");
    $stmt->bind_param("s", $slug);
    $stmt->execute();
    $result = $stmt->get_result();
    $data = $result->fetch_assoc();
}

// 3. Jika user pakai URL lama (dengan ID), redirect ke slug
else if ($id > 0) {
    $stmt = $conn->prepare("SELECT slug FROM berita WHERE id=? LIMIT 1");
    $stmt->bind_param("i", $id);
    $stmt->execute();
    $result = $stmt->get_result();
    $data = $result->fetch_assoc();
    
    // Redirect 301 (permanent) ke URL slug
    if ($data && !empty($data['slug'])) {
        header("Location: /berita/" . $data['slug'], true, 301);
        exit;
    }
}

‚ö†Ô∏è Penting untuk Diingat:

  • Slug harus UNIQUE (tidak boleh ada yang sama)
  • Jika judul berita diubah, slug JANGAN diubah (akan break URL yang sudah di-share)
  • 301 Redirect dari URL lama ke baru WAJIB untuk SEO
  • Slug sebaiknya pendek (max 5-7 kata) agar mudah dibaca

<?php

echo "Updating Database...\n";

// 1. Tambah kolom tagline
$res = $conn->query("SHOW COLUMNS FROM header_settings LIKE 'tagline'");
if ($res->num_rows == 0) {
    $conn->query("ALTER TABLE header_settings 
                  ADD COLUMN tagline VARCHAR(255) 
                  DEFAULT 'Melayani dengan Sepenuh Hati'");
    echo "[OK] Column 'tagline' added.\n";

    echo "[SKIP] Column 'tagline' exists.\n";
}

// 2. Update Menu Dashboard
$res = $conn->query("SELECT id FROM header_menu 
                     WHERE label IN ('Home', 'HOME', 'Beranda')");
if ($res && $row = $res->fetch_assoc()) {
    $id = $row['id'];
    $conn->query("UPDATE header_menu 
                  SET label='DASHBOARD', url='dashboard' 
                  WHERE id=$id");
    echo "[OK] Menu updated to DASHBOARD.\n";
}

echo "Database update complete.\n";

// Upload foto dengan AJAX
btnUploadFoto.addEventListener('click', async () => {
    if (!fotoInput.files || !fotoInput.files[0]) {
        setStatus(uploadStatus, 'Pilih foto terlebih dahulu.', true);
        return;
    }

    const fd = new FormData();
    fd.append('foto', fotoInput.files[0]);

    try {
        const res = await fetch('profile_upload.php', {
            method: 'POST',
            body: fd,
            credentials: 'same-origin'
        });

        const json = await res.json();
        if (!res.ok || json.status !== 'success') {
            setStatus(uploadStatus, json.message || 'Upload gagal.', true);
            return;
        }

        profileImage.src = '../' + json.foto;
        setStatus(uploadStatus, 'Foto berhasil diupload!');
        fotoInput.value = ''; // Reset input
    } catch (err) {
        setStatus(uploadStatus, 'Terjadi error saat upload.', true);
    }
});

3. Perbaikan Backend (profile_upload.php):

<?php
// Validasi upload
if (!isset($_FILES['foto']) || $_FILES['foto']['error'] !== UPLOAD_ERR_OK) {
    echo json_encode(['status' => 'error', 'message' => 'File tidak valid']);
    exit();
}

// Validasi ekstensi
$allowedExt = ['jpg', 'jpeg', 'png', 'webp'];
$ext = strtolower(pathinfo($_FILES['foto']['name'], PATHINFO_EXTENSION));
if (!in_array($ext, $allowedExt)) {
    echo json_encode(['status' => 'error', 'message' => 'Format tidak didukung']);
    exit();
}

// Buat folder jika belum ada
$uploadDir = __DIR__ . '/../uploads/admin';
if (!is_dir($uploadDir)) {
    mkdir($uploadDir, 0777, true);
}

// Upload file
$newName = "admin_" . $adminId . "_" . time() . "." . $ext;
$targetPath = $uploadDir . DIRECTORY_SEPARATOR . $newName;

if (move_uploaded_file($_FILES['foto']['tmp_name'], $targetPath)) {
    // Simpan ke database
    $dbPath = "uploads/admin/" . $newName;
    $stmt = $conn->prepare("UPDATE admin SET foto = ? WHERE id = ?");
    $stmt->bind_param("si", $dbPath, $adminId);
    $stmt->execute();
    
    echo json_encode([
        'status' => 'success',
        'foto' => $dbPath
    ]);
} else {
    echo json_encode(['status' => 'error', 'message' => 'Gagal upload']);
}
